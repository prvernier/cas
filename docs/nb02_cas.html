CAS Attributes

Updated: 03 September 2019

Overview

  • The nb02 database is a merge of the Forest, NonForest, Waterbody, and Wetland shapefiles
  • The attribute “src_filename” identifies which record belongs to which shapefile

Attributes

cas_id

The cas_id attribute is used in 5 of the 6 CASFRI attribute tables and is only described here. It has been modified slightly from the CAS_04 and is a concatenation of several attributes and strings (‘nb02’, src_filename, rc_filename, ,stdlab, ogc_fid).

orig_stand_id

  • The CAS attribute orig_stand_id is created from the FRI attribute stdlab
  • Only Forest has non-zero values; should we have a code for the value 0?
x = mutate(x, orig_stand_id = stdlab)

stand_structure

  • Please confirm the logic here:
    • If src_filename=“geonb_forest-foret” and l2vs=0, then stand_structure=“S”
    • If src_filename=“geonb_forest-foret” and (l1vs>0 and l2vs>0) then stand_structure=“M”
    • If src_filename=“geonb_forest-foret” and (l1vs>1 and l2vs>1) then stand_structure=“C”
  • The variables 11vs and l2vs were used to create stand_structure in the Perl code; here is a crosstabulation:
table(x$l1vs[x$src_filename=="geonb_forest-foret"], x$l2vs[x$src_filename=="geonb_forest-foret"])
##

x = mutate(x, stand_structure = case_when(
    src_filename=="geonb_forest-foret" & l2vs==0 ~ "S",
    src_filename=="geonb_forest-foret" & (l1vs>1 & l2vs>1) ~ "C",
    src_filename=="geonb_forest-foret" & (l1vs>0 & l2vs>0) ~ "M",
    TRUE ~ "NULL_VALUE"))

num_of_layers

  • If src_filename=“geonb_forest-foret” and stand_structure=“S” then num_of_layers=1
  • If src_filename=“geonb_forest-foret” and stand_structure=“C” or “M” then num_of_layers=2
x = mutate(x, 
    num_of_layers = case_when(
        src_filename=="geonb_forest-foret" & stand_structure=="S" ~ 1,
        src_filename=="geonb_forest-foret" & stand_structure %in% c("M","C") ~ 2,
        TRUE ~ -8887))
